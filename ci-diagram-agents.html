System Prompt: 7 Sections ▼
1. Identity
Senior equity research analyst at Continuum Intelligence. First-person plural. Direct, precise, opinionated — like a fund manager talking to another fund manager.
2. Price Driver Awareness
Reference recent 2D/5D/10D performance, broker activity, social signals when relevant. Link current narrative to price moves.

Australian English, no em-dashes. Lead with conclusion. Quantify or cut. Institutional vocabulary ("the print", "the tape", "the multiple"). 150-250 words. Never fabricate.

Hypotheses (ranked by conviction). Skew direction. Position in range. Tripwires (top 6). Discriminators (top 5). Evidence (top 8). Verdict. Narrative text.

Structural / Positional / Tactical memories with age and confidence labels. Lower-scored memories dropped first if over budget.
6. Personalisation (if configured)
Big Five + CRT score. Cognitive profile. Bias counter-interventions. Delivery preferences (verbose vs. terse).
7. Staleness Warning (conditional)
Injected only if research > 30 days old. Signals information recency concern.
Retrieval & Memory ▼
Hybrid retrieval: BM25 + cosine similarity → Reciprocal Rank Fusion (top 10 passages).
Memory extraction (background, Haiku): Max 5 per turn. Types: conviction, risk_flag, valuation, thesis_challenge, process_note. Deduplication at cosine ≥ 0.90.
Conversation persistence: Per-ticker in PostgreSQL. Rolling summarisation at 20 messages. 4000-token sliding window.

Per-Turn: Memory Extraction
Haiku-powered fire-and-forget task after each response. Max 5 memories. Stored in PostgreSQL with version hash.
Nightly: Batch Consolidation
Union-find clustering (cosine ≥ 0.85). Contradiction detection. Guest cleanup (>7d deactivate, >30d hard-delete).
Nightly: Proactive Insights
Scan active memories against fresh research JSON. Haiku classifies: CONFIRMS / CONTRADICTS / NEUTRAL. Re-notification guard: skip if notified within 7 days.

⚡ Should the Analyst & PM Chats Be Autonomous Agents?

Current State: Reactive RAG
Both Analyst and PM chats are reactive systems. They respond to user queries, not self-directed. Users initiate every conversation turn.
Agent-Like Properties
Memory extraction: Runs autonomously after each turn (fire-and-forget Haiku).

Nightly batch jobs: Memory consolidation and proactive insights run without user input.

Pre-LLM analytics: PM runs deterministic alignment and breach detection before generating a response.
What's Missing for Full Agency
Autonomous monitoring: No system watches for price moves or news that affect positions and proactively alerts.

Cross-role coordination: Analyst doesn't auto-flag coverage gaps to PM when portfolio changes.

Self-directed research: PM doesn't autonomously trigger Analyst deep-dives into under-covered stocks.
Recommendation: Augmented RAG + Event Bus
The current "augmented RAG" architecture (memory + retrieval + background tasks) is sound. Converting the chats to true agents would require an event-driven architecture:

• Price/news events: Trigger PM re-evaluation of affected positions.
• Portfolio changes: Trigger Analyst coverage requests for new holdings.
• Evidence changes: Trigger PM reweighting recommendations.

The Batch Analysis and Proactive Insights agents (Agents 5 & 6) are already the closest to true autonomous agents. They run on schedule and push outcomes without user input. An event bus would make Analyst and PM similarly autonomous.
